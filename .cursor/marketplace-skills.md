# Marketplace Skills

## Connected Skills

| # | Skill | Why Connected |
|---|-------|--------------|
| 1 | `next-best-practices` | Enforces App Router patterns, Server Components, and proper data fetching in Next.js |
| 2 | `shadcn-ui` | Guides proper usage of shadcn/ui components, prevents modifying component library files |
| 3 | `tailwind-design-system` | Ensures consistent spacing, colors, and responsive design patterns with Tailwind CSS |
| 4 | `nestjs-best-practices` | Enforces NestJS module structure, dependency injection, and decorator patterns |
| 5 | `prisma-orm` | Covers schema design, migrations, client usage, and performance optimization |
| 6 | `docker-expert` | Guides Docker Compose configuration, multi-stage builds, and container best practices |
| 7 | `postgresql-table-design` | Covers indexing strategies, data types, and schema normalization for PostgreSQL |

## What Custom Skills Cover (Not in Marketplace)

| Custom Skill | Gap Filled |
|-------------|-----------|
| `observability-integration` | Marketplace skills don't cover Signal Lab's specific Prometheus/Loki/Sentry patterns |
| `nestjs-endpoint-scaffold` | Provides project-specific boilerplate with MetricsService and PrismaService integration |
| `prisma-schema-review` | Adds project-specific conventions (cuid IDs, migration workflow) not in generic Prisma skill |

## Usage Notes

- Marketplace skills provide general best practices
- Custom skills provide Signal Lab-specific patterns and conventions
- When conflicts arise, custom skills take precedence (they're scoped to this project)
- Use marketplace skills for learning, custom skills for doing
